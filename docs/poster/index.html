应用价值与社会效益
心血管病管理缺口巨大，患者离院后处于"信息孤岛"
中国心血管病（CVD）患者超 3.3 亿，年死亡超 400 万。70% 患者离院后因缺乏随访而延误干预； 医生日均接诊 30-50 人，无暇主动追踪每一位患者；高危患者病情加重时往往错过最佳干预窗口。 PatientClaw 将 AI Agent 自动化引入诊后管理，打通医患信息断点，让每一次就诊都有后续。

OpenClaw 自动触发 · 全流程
OpenClaw 每天自动扫描患者就诊信息，结合诊断、用药、历史随访记录生成个性化随访内容， 自动解析患者回复并更新病情档案，全程无需人工干预。医生每天只需打开飞书查看日报。
📅 随访全流程
① 读取患者档案
获取诊断（高血压/冠心病/心衰）、用药方案、就诊日期、既往随访记录
触发依据：就诊日期 + 随访策略
↓
② AI 生成个性化随访消息
第3天问基础服药；第7天问效果+症状；第14天总结评估。结合诊断和用药，不使用通用模板
个性化 · 动态生成
↓
③ 发送随访 → 接收患者回复
飞书发送；患者回复（含延迟、不回复）由 OpenClaw 模拟真实行为，含用药偏差和主动提问
本轮回复 67% · 下轮回复 20%
↓
④ AI 解析回复 → 写入结构化数据
提取血压/心率、依从性、病情状态、不良反应 → JSON → followups.csv + chat_logs.csv
↓
⑤ 生成病情报告 + 医生日报
每位患者独立档案；每天汇总所有随访更新、患者提问，生成飞书推送日报
