应用价值与社会效益
心血管病管理缺口巨大，患者离院后处于"信息孤岛"
中国心血管病（CVD）患者超 3.3 亿，年死亡人数超 400 万。70% 患者离院后因缺乏随访而延误干预； 医生日均接诊 30-50 人，无暇主动追踪；高危患者（心衰、高血压 3 级）病情加重时往往错过最佳窗口。 PatientClaw 将 AI Agent 自动化引入诊后管理，打通医患信息断点，让每一次就诊都有后续。

OpenClaw 自动触发 · 全流程详解
OpenClaw 定时调度引擎每天自动扫描患者就诊信息，结合诊断、用药、历史随访记录生成个性化随访内容， 自动解析患者回复并更新病情档案，全程无需人工干预。医生每天只需打开飞书查看日报摘要。
📅 随访全流程（按就诊日期自动触发）
① OpenClaw 读取患者档案
获取患者诊断（高血压/冠心病/心衰等）、当前用药方案（药名+剂量）、就诊日期、既往随访记录
触发依据：就诊日期 + 随访策略
↓
② AI 生成个性化随访消息
第3天问基础服药+初始不适；第7天问用药效果+症状变化；第14天做总结性评估。消息结合患者具体诊断和用药，不使用通用模板
个性化 · 动态生成 · 医嘱要点嵌入
↓
③ 发送随访 → 模拟接收患者回复
通过飞书发送至患者，患者回复（含延迟回复、不回复）由 OpenClaw 模拟真实患者行为，含用药偏差、症状描述、主动提问
本轮回复 67% · 下轮回复 20% · 更晚 13%
↓
④ AI 解析回复 → 写入结构化数据
提取关键信息：血压/心率数值、依从性、病情状态、不良反应。输出 JSON 格式 → followups.csv + chat_logs.csv
自然语言 → 结构化数据
↓
⑤ 生成患者病情报告 + 医生日报
每位患者独立 Markdown 档案（历史记录全部保留）；每天汇总所有随访更新、患者提问、用药效果数据，生成飞书推送日报
患者档案 · 医生日报 · 实时推送
⏰ OpenClaw 定时任务（每天自动执行）
🤖 调度引擎 · 每天自动扫描 · 无需人工干预
每天
检查需随访患者 — 根据每位患者的就诊日期和随访策略（标准/加密/高频），判定哪些患者需要在当前节点接受随访
每天
生成并发送随访消息 — AI 读取患者诊断+用药+历史记录，生成个性化随访提示词，通过飞书发送至患者
实时
监听并解析患者回复 — 监听患者回复，AI 提取关键信息（血压/心率/依从性/症状），写入结构化数据
实时
更新患者病情档案 — 追加最新随访摘要至 Pxxx_report.md，14天随访结束后生成完整终期报告
每天
生成医生综合日报 — 汇总当日所有随访更新，用药效果数据，患者主动提问，生成飞书推送日报
