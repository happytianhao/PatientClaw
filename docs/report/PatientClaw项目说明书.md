# PatientClaw：患者诊后病情全自动跟踪统计系统

**项目说明书** · 北纬·龙虾大赛（第一届）· OpenClaw Hackathon

---

## 一、项目背景与问题定义

### 1.1 医疗随访的现实困境

在基层医疗场景中，患者就诊后的"诊后断联"现象是一个长期被忽视却影响深远的系统性问题。

**患者维度**：大多数患者在取药离诊后不会主动复诊或反馈治疗效果。数据显示，基层门诊的自然复诊率不足30%，意味着超过70%的患者就诊结果对医生而言是完全黑盒的。

**医生维度**：基层全科医生每日接诊量通常在30至50人次，繁重的日常工作使得主动随访几乎不可能实现。即便有意愿，也缺乏系统化工具支持。

**医术成长维度**：临床医术的精进依赖"诊断→治疗→反馈→调整"的闭环。当反馈环节几乎不存在时，医生只能依赖有限的显性案例积累经验，难以系统性地评估自己的用药方案是否最优、诊断思路是否准确。

### 1.2 核心痛点总结

| 痛点 | 影响 |
|------|------|
| 随访工作全靠人力 | 医生没有时间和精力执行 |
| 治疗反馈断链 | 无法积累数据优化用药策略 |
| 依从性不透明 | 无法区分"药不对症"和"患者未遵医嘱" |
| 高危患者缺乏预警 | 可能错过最佳干预时机 |

## 二、解决方案：PatientClaw

### 2.1 核心思路

PatientClaw 是一个基于 **OpenClaw**（本地 AI 智能体框架）构建的患者诊后智能随访系统。其核心思路是：

> **用 AI 全自动承担随访工作，医生只需查看日报，无需任何额外操作。**

系统在医生完全不改变工作习惯的前提下，自动模拟患者随访全流程：生成随访消息、模拟患者回复、解析回复数据，最终将洞察以结构化日报的形式推送给医生。

**角色分工**：

| 角色 | 由谁扮演 | 说明 |
|------|---------|------|
| 医生（李晓峰） | 真实用户 | 只需被动接收飞书日报 |
| 患者（P001-P030） | OpenClaw 模拟 | 自动生成回复，模拟真实延迟和内容 |

### 2.2 工作流程

```
就诊 (Day 0)
  ↓
  Day 1 ─── OpenClaw 生成个性化随访消息（首次随访，就诊后10分钟触发）
                    ↓
            OpenClaw 模拟患者回复 → 解析回复 → 更新病情档案
                    ↓
  Day 3/5 ── 同上（中期随访，30/50分钟触发）
                    ↓
  Day 7/10 ── 同上（70/100分钟触发）
                    ↓
  Day 14 ─── 同上（末期随访，140分钟触发）→ 生成完整版病情追踪报告
                                            ↓
  每10分钟 ── OpenClaw 汇总分析 ──────────→ 医生综合日报（飞书推送）
```

### 2.3 OpenClaw 能力映射

| 功能模块 | 使用的 OpenClaw 能力 |
|---------|-------------------|
| 定时触发随访 | 调度引擎（定时任务） |
| 个性化消息生成 | LLM 推理 + 模板引擎 |
| 飞书消息收发 | 飞书 Bot 集成（Tool Use） |
| 患者回复理解 | LLM NLP 解析 + JSON 结构化输出 |
| 随访数据写入 | 文件操作（CSV 读写） |
| 报告生成 | LLM 生成 + 格式化输出 |

## 三、系统功能详解

### 3.1 智能个性化随访消息

区别于通用模板，PatientClaw 的随访消息由 OpenClaw 结合以下上下文动态生成：
- 患者的具体诊断结果
- 所用药物及服用方式
- 就诊已过天数（Day 1 / 3 / 5 / 7 / 10 / 14 问法不同）
- 本患者的随访历史（避免重复提问）

**示例（王建国，高血压，Day 3）：**
> 您好，王建国先生！我是李晓峰主任医师诊室的智能随访助手。您上次就诊后已经过去3天了，氨氯地平片是否已经开始规律服用？这几天血压自测情况怎么样？有没有出现踝部水肿或头痛加重的情况？请简单告知，医生会及时了解您的情况。

**示例（韩梅，心力衰竭，Day 7）：**
> 韩阿姨您好！距上次李晓峰主任调整您的心衰方案（加用沙库巴曲缬沙坦）已一周了，新方案服用是否顺利？这几天气短和双踝水肿情况有没有改善？每天早晨的体重记录怎么样？您的情况对医生评估用药效果非常重要，期待您的回复。

### 3.2 患者回复智能解析

OpenClaw 从患者的自然语言回复中提取结构化信息：

```json
{
  "medication_adherence": "good",        // 用药依从性
  "condition_status": "improved",        // 病情状态
  "adverse_reaction": "轻微踝部水肿",    // 不良反应
  "needs_followup": true,               // 是否需要进一步跟踪
  "summary": "患者按时服药，血压下降至138/88mmHg，出现轻度踝部水肿，需关注"
}
```

### 3.3 个人病情追踪报告

每位患者在 Day 14 随访后触发生成，包含：
- 基本信息与诊断摘要
- 药方详情
- 三次随访记录时间线
- 用药依从性评估
- 病情进展趋势分析
- 后续建议

### 3.4 医生综合日报（每天）

每天自动生成并推送，采用"重点优先"结构：
1. ⚠️ 需重点关注的患者（未痊愈 / 症状加重 / 依从性差）
2. ✅ 情况良好 / 已痊愈的患者
3. 📊 用药方案效果统计
4. 🔔 建议主动联系或安排复诊的患者

## 四、技术实现

### 4.1 技术栈

| 组件 | 方案 |
|------|------|
| AI 推理与编排 | OpenClaw（本地部署） |
| 消息通道 | 飞书机器人（Feishu Bot API） |
| 数据存储 | CSV 文件（patients / visits / prescriptions / followups）+ Markdown 病情档案 |
| 任务调度 | OpenClaw 内置定时任务（患者询问每30分钟、日报与病情总结每1小时） |

### 4.2 数据模型

**患者基础信息（patients.csv）**：patient_id, name, age, gender, occupation, feishu_account, medical_history, allergies, emergency_contact

**就诊记录（visits.csv）**：visit_id, patient_id, doctor_id, visit_date, chief_complaint, diagnosis, notes

**药方详情（prescriptions.csv）**：prescription_id, visit_id, medication, dosage, frequency, duration_days, instructions

**随访记录（followups.csv）**：followup_id, patient_id, visit_id, followup_day, sent_time, message_sent, reply_received, reply_time, medication_adherence, condition_status, adverse_reaction, needs_followup, ai_summary

### 4.3 核心 Prompt 设计原则

- **随访消息**：温暖专业，结合诊断定制问题，150字以内，引导简短回复
- **回复解析**：输出固定 JSON 结构，不确定项标记 unknown，不猜测
- **医生报告**：重点突出，数据驱动，每条建议可操作

## 五、演示说明

### 模拟数据集

内置 1 名医生 × 30 名患者的完整数据，覆盖高血压、冠心病、心律失常、心力衰竭、高脂血症等心血管科常见疾病场景。

### 演示流程

1. 系统读取病例数据，扫描需随访患者
2. OpenClaw 生成个性化随访消息（展示不同诊断的消息样本）
3. OpenClaw 模拟患者回复（不同依从性和病情状态）
4. OpenClaw 解析回复，写入随访记录
5. 生成 2 份个人病情追踪报告示例
6. 生成 1 份医生综合日报示例

## 六、项目价值与影响

**对医生**：从"经验积累靠感觉"到"数据驱动的临床反思"，低成本获得过去几乎不可能拥有的随访数据。

**对患者**：每位患者感受到被关注，同时获得用药提醒和专业问询，提升治疗依从性和满意度。

**对医疗系统**：随访的自动化降低了医疗资源压力，同时为早期识别高危患者提供了可能。

**可扩展性**：
- 支持接入电子病历系统（替换 CSV 数据层）
- 支持更多消息渠道（微信、短信）
- 支持多医生、多科室模式
- 可积累数据后进行横向用药效果分析
