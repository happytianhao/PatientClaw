# PatientClaw

**患者诊后病情全自动跟踪统计系统 · 助力医生提升医术**

> 北纬·龙虾大赛（第一届）· OpenClaw Hackathon · 生产力赛道

---

## 这是什么

PatientClaw 是一个用 **OpenClaw 龙虾框架**搭建的患者诊后智能随访系统。

医生每天接诊几十名患者，但患者就诊后"一去不回"——药方是否有效、病情是否好转，医生完全不知道。我用龙虾给医生配了一个不知疲倦的数字助理，自动承接随访工作：在正确的时间向患者发出专业问询，理解患者回复，整理成报告推送给医生。

**医生不需要改变任何工作习惯，龙虾在后台静默运行。**

---

## 核心功能

- **全自动随访**：就诊后第 3 / 7 / 14 天，龙虾自动生成并发送个性化随访消息
- **患者回复模拟**：OpenClaw 自动模拟患者回复（含延迟、不回复等真实场景）
- **个性化问询**：结合每位患者的诊断和药方生成专属消息，不是通用模板
- **回复理解**：解析患者的自然语言回复，提取用药依从性、症状变化、不良反应
- **异常预警**：自动识别依从性差、病情加重等问题，标记 attention / urgent
- **个人追踪报告**：为每位患者生成标准化的病情追踪档案
- **医生日报**：每天汇总所有患者情况，重点标注需关注的患者，推送至医生飞书

**医生只需查看飞书日报，系统其余环节全部由龙虾自动运行。**

---

## 快速了解

```
就诊 Day 0
  ↓
Day 3  → 龙虾生成随访消息 → 模拟患者回复 → AI 解析记录 → 更新病情档案
Day 7  → 中期随访（同上）
Day 14 → 末期随访（同上）→ 生成个人病情追踪报告
每10分钟 → 生成医生综合日报 → 飞书推送给医生
```

时间压缩模型：**系统内 10 分钟 = 现实 1 天**，演示时可在约 2.5 小时内跑完完整随访周期。

## 演示数据

| 指标 | 数值 |
|------|------|
| 患者总数 | 30 人（心血管科） |
| 随访记录 | 50 条 |
| 完整日报 | 10 天 |
| 病情追踪报告 | 30 份 |
| 整体用药依从率 | 约 92% |

病种覆盖：高血压（13人）· 冠心病（8人）· 心律失常（4人）· 心力衰竭（5人）

## 技术栈

| 组件 | 方案 |
|------|------|
| AI 推理与编排 | OpenClaw（本地部署） |
| 消息通道 | 飞书机器人 |
| 数据存储 | CSV 文件 + Markdown 病情档案 |
| 任务调度 | OpenClaw 定时任务（每 10 分钟） |
